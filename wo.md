## 1. 综合业务管理系统 BMS

技术栈：Vue2 + VueRouter + Monorepo + Wujie + Rsbuild

项目背景：
公司各个业务线存在独立后台系统，各自重复开发用户登录、用户信息、角色管理、部门管理等基础功能，造成研发资源浪费且权限模型不统一，存在未授权访问风险。

解决方案：

1. 基于 RBAC 实现动态路由鉴权，通过 require.context 页面文件解析，结合后端返回的菜单数据动态注册路由，确保无权限页面无法访问；
2. 采用 pnpm Monorepo 架构拆分主项目与子业务系统，统一依赖版本，实现跨项目原子提交，解决多团队协作冲突；
3. 集成 Wujie 微前端方案，定义标准化子应用接入协议，降低接入成本 80%；
4. 针对多子应用并行开发卡顿问题，使用 Rsbuild 代替 Webpack 方案，提升开发体验；

项目成果：

1. 100% 覆盖公司内部后台系统，单个项目节省 10 人日
2. 权限模型标准化，安全漏洞归零，测试回归效率提升 40%

## 2. 文件上传服务

技术栈：Express + aliyun OSS + nacos

项目背景：

各业务系统独立对接阿里云 OSS，开发接入成本高且难以集中管控。

解决方案：

1. 基于 Express 构建独立 Node.js 中间层，统一封装 OSS API，业务系统快速集成
2. 集成 Nacos 配置中心，按 appId 配置各业务系统的 Bucket、权限策略，支持运行时热更新，无需重启服务；
3. 采用 formidable 流式解析替代内存缓存，降低 70% 内存峰值
4. 基于 p-queue 实现大小文件分级队列，大文件限流保护带宽，小文件优先处理保障体验

项目成果：

1. 作为公司文件上传的基础服务能力，支撑单日 10w+ 请求；
2. OSS 凭据集中管理，有效管理 AK/SK；

## 3. 多源认证 SaaS 服务

技术栈：Vue3 + FormKit/Zod + Module Federation + Strapi

项目背景：
公司需为金融、政务等客户快速提供合规身份认证能力（公安网证、支付宝实名、人脸识别等），但面临认证流程组合多样化多变、开发强依赖真机 App 调试。

解决方案：

1. 基于 FormKit 维护 schema 动态渲染组件，并集成 Zod 校验库，提升数据准确性
2. 针对不同应用在 Strapi 配置认证流程组合方式，同时维护认证协议、页面元信息，业务人员可自助调整流程，无需研发介入；
3. 基于 Node.js 自研 Socket 代理服务，在浏览器中模拟 JSBridge 环境，实现 PC 端 100% 覆盖 App SDK 流程调试，开发效率提升 10 倍；
4. 通过 Module Federation 将认证组件作为远程模块暴露，按配置加载对应版本，线上问题即时修复。

项目成果：

1. 支撑每月百万认证页面调用；
2. 客户定制需求交付周期从 1 周缩短至 21 天；
3. 调试基建推广公司其他混合应用，节省调试成本 50%；.
4. 动态表单体系减少 90% 硬编码，认证流程测试冒烟成功率提升。

## 4. AI 低代码后台生成平台

技术栈：Vercel AI SDK + DeepSeek + amis + Next.js

项目背景：

公司内部存在大量 CRUD 型后台管理系统，传统开发模式前后端协作开发效率低，还需要测试资源，受 next-ai-draw-io 启发，决定构建 “自然语言 → 可运行后台页面” 的 AI 低代码平台。

解决方案：

1. 基于 Next.js + Vercel AI SDK 搭建对话式前端页面，调用 DeepSeek 模型，生成符合 amis 标准的 JSON Schema
2. 生成的 amis Schema 可直接被现有后台系统加载，零改造接入
3. 对高频场景（如搜索表单、详情页）建立 prompt 模板库，提升准确率，降低返工。

项目成果：

1. 缩短开发周期，生成的 amis 配置直接投入上线
2. 推动 AI 技术在开发场景的应用，作为公司内部 AI 试用项目的成功案例
